# CMC风险管理与可比性评估
参考 playbook。用于变更风险评估、可比性方案设计、PACMP/EC 工具应用。

## 概述

本 playbook 提供**生物药 CMC 风险管理与可比性评估**的系统化框架，帮助在工艺变更、场地转移、供应商切换等场景下做出科学的可比性论证。

> 核心理念：风险管理不是"消灭一切风险"，而是**将残留不确定性降低到监管可接受的水平**。

这意味着：
- 你不需要证明"变更前后完全一样"（事实上不可能完全一样）
- 你需要证明"差异不构成本质性影响"（material impact）
- 论证的深度取决于风险的级别——高风险变更需要更多证据
- 可比性是一个**分层论证**的过程：能在线解决的不上临床，能非临床解决的不上临床

## 第一部分：风险管理体系

### 1.1 风险台账（Risk Register）

风险台账是所有变更管理的起点。每个变更必须先登记，再评估，最后归档。

#### 六大风险类别

| 类别 | 覆盖范围 | 典型风险举例 | 常见变更触发 |
|------|---------|-------------|-------------|
| **产品属性** | 结构/功能表征、质量属性 | 糖型偏移、电荷变异体升高、聚体增加 | 细胞株变更、工艺参数调整 |
| **工艺** | 上游培养、下游纯化、制剂灌装 | 收率下降、杂质清除不充分、工艺窗口收窄 | 工艺优化、规模放大、参数调整 |
| **分析** | 分析方法、稳定性方案、放行标准 | 方法专属性不足、方法变更导致结果漂移 | 方法升级/替换、新杂质发现 |
| **设施设备** | 厂房、设备、公用系统 | 交叉污染、设备性能偏移、环境监测超标 | 场地转移、设备替换、扩建 |
| **供应链** | 原材料、耗材、关键供应商 | 供应商停产、物料质量波动、断供 | 供应商变更、原材料替换 |
| **数据完整性** | 记录、审计追踪、电子数据 | 数据篡改、记录不完整、审计追踪缺失 | 系统升级、新LIMS/MES上线 |

### 1.2 FMEA 风险分析

采用 FMEA（Failure Mode and Effects Analysis）对每个识别的风险进行量化评估。

#### 评分标准

| 维度 | 评分 | 定义 |
|------|------|------|
| **严重度 S** (Severity) | 1 | 对产品质量无影响 |
| | 2-3 | 轻微影响，在历史变异范围内 |
| | 4-6 | 中等影响，可能超出历史范围但不影响安全性和有效性 |
| | 7-8 | 严重影响，可能影响安全性和有效性 |
| | 9-10 | 灾难性影响，直接导致产品不合格或患者安全风险 |
| **发生概率 O** (Occurrence) | 1 | 极不可能发生 |
| | 2-3 | 低概率，偶发 |
| | 4-6 | 中等概率 |
| | 7-8 | 高概率，经常发生 |
| | 9-10 | 几乎必然发生 |
| **可检测性 D** (Detectability) | 1 | 现有方法极易检测到 |
| | 2-3 | 现有方法能可靠检测 |
| | 4-6 | 有一定检测能力但不够灵敏 |
| | 7-8 | 检测困难 |
| | 9-10 | 几乎无法检测 |

#### RPN 计算

```
RPN = S x O x D
RPN 范围：1 - 1000

风险等级判定：
  RPN ≤ 30  → 低风险（常规监控即可）
  31 ≤ RPN ≤ 100 → 中风险（需要缓解措施）
  RPN > 100 → 高风险（必须采取措施，可能需要可比性研究）
```

#### FMEA 评估模板

| 字段 | 内容 |
|------|------|
| 风险编号 | R-2026-XXX |
| 失败模式 | [可能出什么问题] |
| 失败影响 | [出问题后会有什么后果] |
| 失败原因 | [为什么会出问题] |
| 现有控制措施 | [目前有什么手段防止或检测] |
| S 评分 | [1-10] |
| O 评分 | [1-10] |
| D 评分 | [1-10] |
| RPN | S x O x D |
| 建议措施 | [降低风险的具体行动] |
| 措施后 RPN | [实施措施后的重新评分] |

## 第二部分：可比性三步法

可比性论证遵循**阶梯式递进**原则：优先用最少的证据证明可比，只有当低层级证据不足时才进入下一层。

### Step 1：质量层级对比（Quality-Level Comparison）

> 大多数变更（约 80%）可以在此步骤完成论证。

#### 分析可比性（Analytical Comparability）

**必须做的检测项目：**

| 检测类别 | 具体方法 | 目的 |
|---------|---------|------|
| **一级结构** | 肽图、LC-MS、测序 | 确认氨基酸序列未变 |
| **高级结构** | CD光谱、DSC、HDX-MS | 确认折叠和构象未变 |
| **翻译后修饰** | 糖型分析（HILIC/CE）、氧化、去酰胺化 | PTM谱可比 |
| **电荷变异体** | CEX、IEF | 酸/碱变异体分布可比 |
| **大小变异体** | SEC、CE-SDS（还原/非还原） | 聚体、片段水平可比 |
| **效价** | 细胞-based assay或结合ELISA | 生物活性可比 |
| **杂质谱** | HCP、DNA、Protein A、内毒素 | 杂质清除能力未降级 |
| **稳定性** | 加速、强制降解 | 降解路径和速率可比 |

#### 验收标准设定原则

| 原则 | 说明 | 示例 |
|------|------|------|
| **预定义** | 必须在实验开始前以书面形式定义 | 可比性方案中明确写入 |
| **基于历史数据** | 使用变更前历史批次统计确定范围 | mean +/- 3SD 或 tolerance interval |
| **分层设定** | 关键质量属性收紧，非关键属性可适当放宽 | 效价：90-110%；HCP：<2倍 |
| **统计学合理** | 使用合适的统计方法 | 等价检验（equivalence test）而非简单t检验 |

#### 功能可比性（Functional Comparability）

对于有已知 MoA（作用机制）的产品，需要额外论证功能层面的可比性：

| MoA 类型 | 功能实验 | 关注点 |
|---------|---------|--------|
| ADCC | Fc受体结合 + 细胞杀伤 | Fc功能未受影响 |
| CDC | 补体激活实验 | 补体通路完整 |
| FcRn结合 | SPR/BLI | 半衰期相关功能未变 |
| 受体阻断 | 竞争结合实验 | 阻断活性可比 |
| Apoptosis诱导 | Annexin V/PI染色 | 细胞死亡诱导能力可比 |

### Step 2：非临床对比（Non-Clinical Comparison）

> 当质量层级数据出现**超出预设标准的差异**，但无法判断是否影响安全性和有效性时触发。

#### PK/PD 对比

| 项目 | 内容 |
|------|------|
| **触发条件** | FcRn结合变化、糖型显著偏移、可能影响清除率的变更 |
| **实验设计** | 交叉设计或平行设计，n>=6/组，通常用啮齿类或非人灵长类 |
| **关键参数** | AUC、Cmax、Tmax、t1/2、CL |
| **验收标准** | 90% CI 在 80%-125% 范围内（与生物类似物标准一致） |

#### 毒理学对比

| 项目 | 内容 |
|------|------|
| **触发条件** | 杂质谱变化（新杂质出现）、工艺变更可能引入新杂质 |
| **实验设计** | 单次/重复给药毒性研究，通常 28 天 |
| **关注终点** | 临床观察、体重、血液学、组织病理学 |

#### 免疫原性对比

| 项目 | 内容 |
|------|------|
| **触发条件** | 聚体升高、新降解产物、制剂重大变更 |
| **实验设计** | 动物抗药抗体（ADA）检测 + 中和抗体（NAb）检测 |
| **局限性** | 动物模型预测人免疫原性能力有限，仅作参考 |

### Step 3：临床桥接（Clinical Bridge Study）

> 当质量层级和非临床层级都无法充分论证可比性时，需要临床数据支持。
> 这是**最后手段**，大多数变更不会走到这一步。

#### 触发条件

- 效价差异超过预定义范围（如效价<80%或>120%）
- 非临床PK参数不等效
- 新杂质无法通过非临床毒理学充分表征
- 监管机构明确要求

#### 典型研究设计

```
设计类型选择：

变更规模          研究设计              样本量
─────────────────────────────────────────────
中变更（PK差异）   单剂量交叉PK研究       30-50人
中变更（PD差异）   多剂量PK/PD研究       50-80人
大变更（效价差异）  临床等效性研究        100-300人
                  （含免疫原性终点）
```

#### 关键设计要素

| 要素 | 建议 |
|------|------|
| **人群选择** | 选择最敏感的人群（通常不是健康志愿者） |
| **剂量选择** | 选择能检测差异的剂量（不一定是临床剂量） |
| **采样点** | 覆盖完整的 PK 曲线（吸收相+分布相+消除相） |
| **免疫原性** | 必须包含 ADA/NAb 监测 |
| **统计方法** | 等价检验，90% CI 在 80%-125% |

## 第三部分：实战案例

### 案例 1：糖型分布偏移（Glycan Shift Within Spec）

**背景：**
某 IgG1 单抗，上游工艺优化后（提高灌流速率），G0F 从 45% 降到 38%，G1F 从 30% 升到 36%，G2F 从 15% 升到 18%。所有糖型均在历史放行范围内，但分布明显偏移。产品仍处于 Phase II 阶段。

**风险评估（FMEA）：**

| 项目 | 评分 | 理由 |
|------|------|------|
| 严重度 S | 5 | 糖型变化可能影响 Fc 功能（ADCC/CDC），但不影响抗原结合 |
| 发生概率 O | 6 | 工艺优化已确认会产生此偏移 |
| 可检测性 D | 2 | 糖型分析是常规检测，容易检测 |
| RPN | 60 | 中风险，需要缓解措施 |

**可比性策略：**

1. **质量层级：**
   - 25 批变更前后产品并排分析（全部 CQA）
   - Fc 受体结合动力学（FcRIIIa、FcRn）→ 结果：FcRIIIa 结合 Kd 变化 <8%，在历史范围内
   - ADCC 效价 → 结果：相对效价 95%，在 80-120% 验收标准内
   - 加速稳定性对比 → 结果：降解曲线重合

2. **结论：** 质量层级论证充分，无需进入非临床。糖型偏移是统计学显著但不具有生物学意义的变化。

**关键教训：**
- "统计学显著"不等于"生物学显著"——要有功能数据支撑
- 糖型分布偏移是最常见的变更后变化之一，提前做好预期管理
- 使用足够多的批次（建议 >=10 批变更前 + >=10 批变更后）进行统计对比

### 案例 2：IV 转 SC 高浓度粘度问题

**背景：**
某生物药从 IV（50 mg/mL）转为 SC 给药，目标浓度 150 mg/mL。浓缩后粘度从 5 cP 飙升至 85 cP，远超 20 cP 的可注射上限。同时观察到可溶性聚体从 0.8% 升至 3.2%。

**风险评估（FMEA）：**

| 项目 | 评分 | 理由 |
|------|------|------|
| 严重度 S | 8 | 高粘度导致无法注射，聚体升高增加免疫原性风险 |
| 发生概率 O | 7 | 已确认发生 |
| 可检测性 D | 3 | 粘度和聚体均能可靠检测 |
| RPN | 168 | 高风险，必须立即采取行动 |

**解决路径：**

1. **制剂处方优化（3 轮筛选）：**
   - 第 1 轮：筛选 pH 5.0-6.5 范围，发现 pH 5.8 最优（粘度降至 55 cP）
   - 第 2 轮：添加精氨酸（50-200 mM），200 mM 时粘度降至 18 cP
   - 第 3 轮：确认 150 mM 精氨酸 + pH 5.8 + 组氨酸缓冲液为最优处方（粘度 16 cP，聚体 1.1%）

2. **可比性论证：**
   - 新旧处方的全面质量对比（结构、功能、稳定性）
   - 12 个月实时稳定性 + 6 个月加速稳定性
   - 精氨酸添加不引入新杂质，不影响效价
   - 功能实验：结合亲和力、Fc功能均无变化

3. **监管沟通：**
   - 由于涉及处方变更 + 给药途径变更，主动与监管沟通
   - 提供可比性方案（Protocol）供审评

**关键教训：**
- IV 转 SC 不是简单的"浓度提高"，而是**全新的制剂开发挑战**
- 粘度和聚体是高浓度制剂的两大核心风险——必须早期评估
- 处方变更 + 给药途径变更的组合是高风险变更，需要充分的稳定性数据

## 第四部分：PACMP 与 EC 工具

### 4.1 PACMP（Prior Approval CMC Post-Approval Change Management Protocol）

#### 定义

PACMP 是一种**预先与监管机构达成一致的可比性方案**，在上市后变更发生时，按照已批准的方案执行即可，无需逐一报批。

#### PACMP 的价值

| 价值维度 | 说明 |
|---------|------|
| **速度** | 变更实施时间从 6-12 个月缩短至 1-3 个月 |
| **确定性** | 提前锁定验收标准，消除审评不确定性 |
| **资源** | 减少重复的监管沟通和补充资料准备 |
| **合规** | 方案经监管批准，执行有据可依 |

#### PACMP 提案结构（6 个核心章节）

| 章节 | 内容 | 关键要点 |
|------|------|---------|
| **1. 变更描述** | 明确变更范围、触发条件、适用产品 | 边界要清晰，不能"留口子" |
| **2. 支持性数据** | 现有产品质量数据、工艺理解、历史变异 | 数据越充分，获批概率越高 |
| **3. 可比性方案** | 检测项目、验收标准、统计方法、批次要求 | 这是 PACMP 的核心——标准必须量化且可操作 |
| **4. 稳定性方案** | 稳定性研究设计、检测频率、承诺 | 通常要求至少 6 个月加速 + 12 个月长期 |
| **5. 实施条件** | 什么情况下可以按照 PACMP 执行 | 明确前置条件和排除条件 |
| **6. 报告承诺** | 变更执行后的报告要求 | 年度报告 or 补充申请，明确写入 |

#### PACMP 适用场景

- 适合：成熟的、高频的、可预测的变更（如培养基供应商变更、层析树脂批间替换）
- 不适合：首次发生的、影响范围不确定的重大变更

### 4.2 EC（Established Conditions）已确定条件

EC 是 ICH Q12 引入的概念，将产品的"已确定条件"分层管理，不同层级对应不同的变更管理要求。

#### EC 三个层级

| 层级 | 定义 | 变更管理 | 示例 |
|------|------|---------|------|
| **EC1** | 对产品质量有关键影响的条件 | 任何变更都需要**事先批准**（Prior Approval） | 灭菌工艺参数、病毒灭活步骤参数、成品配方 |
| **EC2** | 对产品质量有重要但可控影响的条件 | 变更需**通知监管**（Notification），可能需要补充资料 | 层析柱操作 pH 范围、灌装速度范围、中间体保持时间 |
| **EC3** | 对产品质量影响较小或有充分控制的条件 | 变更可在**企业内部管理**，年报中说明 | 培养基储存条件、一般设备参数、非关键分析方法参数 |

#### EC 分层决策逻辑

```
该条件是否直接影响关键质量属性（CQA）？
│
├── 是 → 该条件的变更是否已通过 PACMP 管理？
│        │
│        ├── 是 → 按 PACMP 执行
│        │
│        └── 否 → 该条件是否在获批资料中有具体数值/范围？
│                 │
│                 ├── 是 → EC1（Prior Approval）
│                 │
│                 └── 否 → EC2（Notification）
│
└── 否 → 该条件是否间接影响 CQA（通过工艺控制）？
         │
         ├── 是 → EC2（Notification）
         │
         └── 否 → EC3（内部管理）
```

#### EC 管理的实操建议

1. **建立 EC 清单**：在产品获批时（或补充申请时），明确列出所有 EC 及其层级
2. **定期评审**：每年评审 EC 分类的合理性，随工艺理解的深化调整层级
3. **变更映射**：每个变更必须映射到受影响的 EC，确定变更路径
4. **记录留存**：即使是 EC3（内部管理），也必须有完整的变更记录

## 第五部分：常见变更场景风险评估

### 场景 1：层析树脂供应商变更

**场景描述：** Protein A 层析树脂从 Cytiva（MabSelect SuRe）切换至 Agilent（MAbSelect PrismA），或同品牌不同型号替换。

**风险分析：**

| 评估项 | 风险等级 | 关键风险点 |
|--------|---------|-----------|
| 产品质量 | 中 | 配基密度差异可能影响结合容量和洗脱行为 |
| 杂质清除 | 中-高 | 不同树脂的 HCP、DNA 清除效率不同 |
| 工艺性能 | 中 | 动态结合容量（DBC）变化影响上样量 |
| 分析可比性 | 低 | 不涉及分析方法变更 |

**可比性策略：**

1. **小试确认**（3-5 批次）
   - DBC 对比（10% 穿透点）
   - 洗脱峰形对比
   - HCP/DNA 清除对比
   - 收率对比
2. **中试验证**（2-3 批次）
   - 放大至中试规模，确认工艺性能
3. **可比性批次**（>=3 批）
   - 新树脂生产批次的全面质量属性与历史数据对比
   - 验收标准：基于历史统计范围

**监管路径：** 通常为 CBE-30 或年报（取决于分类），建议提前与监管沟通。

### 场景 2：一次性系统（SU）切换

**场景描述：** 从不锈钢生物反应器切换至一次性生物反应器，或一次性储液袋/管路供应商变更。

**风险分析：**

| 评估项 | 风险等级 | 关键风险点 |
|--------|---------|-----------|
| 可提取物/浸出物 | 高 | 新材料的 E&L 谱不同，可能引入新杂质 |
| 产品质量 | 中 | 混合效率、传质差异可能影响培养 |
| 工艺性能 | 中 | 规模限制（目前 SU 最大 ~2000L） |
| 供应链 | 中 | SU 供应商集中度高，断供风险 |
| 数据完整性 | 低 | 不直接影响 |

**可比性策略：**

1. **E&L 风险评估**（必须做）
   - 新材料的全面可提取物研究
   - 毒理学评估（TTC 方法）
   - 浸出物迁移研究（在产品缓冲液中）
2. **工艺性能确认**
   - 培养 KPI 对比（细胞密度、活率、表达量）
   - 收率对比
3. **产品质量全面对比**
   - 全项放行检测 + 扩展表征
   - 稳定性对比
4. **长期承诺**
   - 供应商审计
   - 双供应商策略（建议）

### 场景 3：生产场地转移

**场景描述：** 将产品从 Site A 转移至 Site B（可能是新厂房或 CDMO）。

**风险分析：**

| 评估项 | 风险等级 | 关键风险点 |
|--------|---------|-----------|
| 设备差异 | 高 | 不同设备尺寸/型号的工艺参数需要重新确认 |
| 人员/操作 | 高 | 新操作团队的经验和 SOP 执行差异 |
| 环境差异 | 中 | 水系统、HVAC、洁净区等级 |
| 供应链 | 中 | 新场地可能使用不同供应商 |
| 监管 | 高 | 不同场地的 GMP 合规状态、批准历史 |

**可比性策略（最全面的方案）：**

1. **前期准备**
   - 差距分析（Gap Analysis）：两个场地的设备、公用系统、SOP 对比
   - 风险评估：针对每个差异点进行 FMEA
   - 可比性方案撰写并（建议）与监管沟通

2. **工艺验证**
   - 技术转移批次（3-5 批）
   - 工艺性能确认（PPQ，3 批）
   - 持续工艺确认（CPV）

3. **产品质量对比**
   - 全项放行 + 扩展表征（>=10 批新场地产品 vs >=10 批原场地历史数据）
   - 统计学对比（等价检验）
   - 稳定性：至少 6 个月加速 + 12 个月长期实时

4. **监管申报**
   - 中国：补充申请（根据变更分类）
   - FDA：Prior Approval Supplement
   - EMA：Type II Variation
   - 建议：多场地同时获批（如果涉及多地上市）

## FAQ

### Q1: 可比性需要多少批次才算"够"？

**A:** 没有绝对标准，但行业共识如下：

| 变更级别 | 变更后批次建议 | 变更前参考批次 |
|---------|--------------|-------------|
| 轻微变更（EC3 级别） | 1-3 批 | >=5 批历史数据 |
| 中等变更（EC2 级别） | 3-5 批 | >=10 批历史数据 |
| 重大变更（EC1 级别） | >=5 批 | >=10 批历史数据 + 统计分析 |

**关键原则：**
- 批次间变异性比批次数量更重要——如果工艺非常稳定，少量批次也能说明问题
- 统计学方法（如 tolerance interval）可以帮助确定最少批次
- 务必使用**变更前同一分析平台**的历史数据作为对比基准

### Q2: 质量属性有差异但仍在放行标准内，算可比吗？

**A:** 不一定。需要分层判断：

- **在放行标准内 + 在历史变异范围内** → 可比，无需额外论证
- **在放行标准内但超出历史变异范围** → 需要功能数据论证（效价、Fc功能等），证明差异不具有生物学意义
- **在放行标准内但呈系统性偏移趋势** → 需要调查根因 + 评估长期影响，可能需要稳定性数据支持

**核心逻辑：** 放行标准是底线，历史变异范围才是"正常"的参照。放行标准是宽泛的"不坏"标准，而可比性需要论证的是"没变"。

### Q3: 什么时候需要主动跟监管沟通可比性方案？

**A:** 建议在以下情况下主动沟通：

1. **首次使用 PACMP** → 提交 PACMP 提案前沟通
2. **重大变更**（场地转移、新厂房）→ 变更实施前沟通方案
3. **质量数据出现意料之外的差异** → 执行过程中及时沟通
4. **跨区域上市** → 不同监管机构可能要求不同，提前对齐
5. **创新技术/没有先例的变更** → 如连续工艺引入、新型一次性技术

**沟通方式：**
- FDA：Pre-Submission Meeting 或 Type B Meeting
- EMA：Scientific Advice 或 Protocol Assistance
- NMPA：沟通交流会
