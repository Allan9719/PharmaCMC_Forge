# CMC Project Archetypes

For Claude Code

Use this file to decide whether a CMC task is a strong fit for Claude Code plus Skill-based project execution.

## Best-Fit Archetypes

### 1. Stage-Gated CMC Readiness

Use for:
- IND-enabling package planning
- Phase I to II transition planning
- Pre-PPQ or pre-BLA readiness reviews
- Gap assessment against fit-for-phase expectations

Why this fits:
- The work is document-heavy and cross-functional.
- It depends on explicit evidence mapping, not only raw experiments.
- Claude Code is good at converting scattered inputs into a coherent readiness package.

Typical inputs:
- Development stage
- Modality and dosage form
- Existing studies and draft reports
- Open risks and change history

Typical outputs:
- Readiness checklist
- Gap log
- Priority matrix
- Evidence package outline
- Meeting briefing note

### 2. Developability And Early Prioritization

Use for:
- Candidate ranking
- Go or no-go discussion support
- Early CMC risk framing
- Resource triage under time pressure

Why this fits:
- The task benefits from structured criteria and transparent assumptions.
- Claude Code can keep decision logic visible and reusable.

Typical outputs:
- Red-yellow-green matrix
- Candidate comparison table
- Decision memo
- "must solve now" versus "can defer" split

### 3. Analytical Lifecycle And Control Strategy Planning

Use for:
- Method portfolio mapping
- Phase-appropriate method plans
- Specification and CQA evidence packaging
- Method upgrade sequencing

Why this fits:
- Analytical work creates many linked documents, owners, and milestones.
- Claude Code can organize lifecycle dependencies even when the science stays with SMEs.

Typical outputs:
- Method lifecycle map
- Evidence matrix
- method-by-method action tracker
- control strategy narrative scaffold

### 4. Technology Transfer And Site Transfer

Use for:
- Internal tech transfer packages
- CDMO transfer planning
- Receiving-site readiness checks
- knowledge-transfer and action registers

Why this fits:
- Transfer work is coordination-heavy and template-friendly.
- Claude Code can keep process knowledge, assumptions, and open actions aligned.

Typical outputs:
- Transfer charter
- transfer package outline
- open issue log
- comparability and validation dependency map
- meeting agenda and decision log

### 5. Change Control And Comparability Planning

Use for:
- Process changes
- raw material or supplier changes
- facility or equipment changes
- post-approval change preparation

Why this fits:
- The key work is risk framing, impact logic, and test-plan drafting.
- Claude Code can turn diffuse change discussions into a structured impact package.

Typical outputs:
- Change impact summary
- risk register
- comparability strategy draft
- PACMP discussion package
- authority interaction questions

### 6. Module 3 And Quality-Package Structuring

Use for:
- Module 3 skeletons
- content inventories
- source-to-section mapping
- QOS and quality narrative planning

Why this fits:
- Submission work involves structure, traceability, and consistency.
- Claude Code is valuable for scaffolding and cross-reference checks.

Typical outputs:
- section map
- missing-content log
- source document tracker
- authoring workplan

### 7. CMC Training, FAQ, And Knowledge Systems

Use for:
- onboarding materials
- internal playbooks
- lessons learned libraries
- reusable FAQ or red-flag lists

Why this fits:
- Training and knowledge work need structure, examples, and repeatability.
- Claude Code can transform expert content into scalable learning assets.

Typical outputs:
- training outline
- FAQ bank
- decision-tree summary
- workshop material

## Medium-Fit Archetypes

These can work well if the user wants drafting and project structure, not final regulated execution.

### 8. Deviation, Investigation, CAPA, Or Inspection Readiness Drafting

Good for:
- draft narratives
- chronology reconstruction
- issue categorization
- question banks for mock inspections

Needs caution because:
- these documents often enter validated quality systems
- facts must be source-backed
- final ownership must remain with authorized roles

### 9. Stability Program Planning And Data Summarization

Good for:
- study matrix planning
- pull-point schedules
- shelf-life decision discussion packs
- trend summary drafts

Needs caution because:
- raw statistical interpretation and final shelf-life claims require SME and QA review
- source data integrity matters

## Weak-Fit Or Off-Limits Areas

### 10. Direct Experimental Execution

Do not treat Claude Code as the operator for:
- running assays
- setting instrument methods without review
- generating GMP batch data
- signing off wet-lab execution decisions

### 11. Final Quality Disposition And Regulatory Authority

Do not treat Claude Code as the final decision maker for:
- batch release
- deviation approval
- CAPA approval
- final regulatory filing signoff
- legal interpretation of regulations

## Quick Selection Rule

Ask:
- Is the task mostly about structuring, planning, mapping, drafting, or reviewing evidence?
- Is the output a package, template, memo, checklist, tracker, or outline?
- Can the human still review before anything enters a controlled system?

If the answer is yes, this is probably a good fit.
