# GxP Boundaries For The CMC Skill

Use this file whenever the task touches regulated quality work.

## Safe Uses

The skill can safely help with:
- Drafting non-final working documents
- Structuring project plans and readiness reviews
- Building risk registers, issue logs, and evidence matrices
- Turning source material into training or knowledge assets
- Preparing question lists for meetings and inspections
- Creating draft templates for later controlled entry

## Use With Explicit Caution

The skill may assist, but must label outputs as drafts, for:
- Change controls
- deviations and investigations
- CAPA narratives
- method validation or qualification summaries
- stability summaries
- tech transfer reports
- Module 3 text drafts
- meeting packages for regulatory interactions

For these, always:
- distinguish facts from assumptions
- cite the source materials provided by the user
- state what still requires SME, QA, RA, or QP review

## Off-Limits As Final Authority

Do not let the skill act as final authority for:
- approving or rejecting product release
- declaring regulatory compliance as a legal fact
- entering controlled records directly into validated systems without user intent
- assigning final root cause without evidence
- concluding comparability or validation success without source data
- signing or substituting for electronic signatures

## Draft Labeling Rule

If the output could later enter a regulated workflow, add a visible label near the top:

`Draft for controlled review. Verify against source data, approved procedures, and the validated system of record before use.`

## Evidence Rule

If the user provides no primary support, do not invent:
- batch values
- acceptance criteria
- study outcomes
- trend conclusions
- authority agreements
- submission commitments

Instead provide one of these:
- a draft structure
- a data-request list
- a review checklist
- a decision framework with blanks to fill

## Human Owners To Call Out

When relevant, explicitly mention one or more of:
- CMC lead
- analytical lead
- process development lead
- MSAT or tech transfer lead
- QA
- RA
- QP
- statistician
- site quality or site operations owner

## Decision Hygiene

When the task is sensitive, end with:
- What is known
- What is assumed
- What must be verified in the validated system
- Who must review before use
