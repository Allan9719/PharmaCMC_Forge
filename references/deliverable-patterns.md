# CMC Deliverable Patterns

Use these patterns to choose the right artifact quickly.

## Pattern 1: Readiness Pack

Use when:
- the user asks "还缺什么"
- the team is moving to the next phase
- a meeting or submission gate is approaching

Recommended sections:
1. Objective and stage
2. Scope
3. Available evidence
4. Critical gaps
5. Risks and dependencies
6. Priority actions
7. Owners and deadlines

## Pattern 2: Risk Register

Use when:
- the task involves changes, transfers, suppliers, scale-up, or timeline compression

Recommended columns:
- Risk
- Category
- Severity
- Likelihood
- Detectability or visibility
- Current controls
- Proposed action
- Owner
- Due date
- Residual risk note

## Pattern 3: Comparability Or Change Package

Use when:
- the user is evaluating a process or site change
- the team needs to explain why the change matters or does not matter

Recommended sections:
1. Change description
2. Product and process impact hypothesis
3. CQA and CPP relevance
4. Proposed comparability evidence
5. Additional nonclinical or clinical bridge considerations
6. Regulatory reporting path assumptions
7. Open questions

## Pattern 4: Tech Transfer Package

Use when:
- the user is handing work to a site or CDMO
- the team needs an execution-ready transfer draft

Recommended sections:
1. Scope and target state
2. Product and process summary
3. Analytical package summary
4. Critical process knowledge
5. Materials and equipment dependencies
6. Risks and assumptions
7. Transfer milestones
8. Acceptance and review gates

## Pattern 5: Module 3 Workplan

Use when:
- the user wants to organize submission writing
- the team has source documents but not a dossier structure

Recommended sections:
1. Section map
2. Source document inventory
3. Missing content by section
4. Author and reviewer assignment
5. Evidence still needed
6. Consistency checks

## Pattern 6: Training Or Playbook Pack

Use when:
- the user wants repeatable internal knowledge transfer
- the team keeps answering the same questions

Recommended sections:
1. Audience and objective
2. Core concepts
3. Decision rules
4. Common failure modes
5. Worked examples
6. FAQ
7. Quick-reference checklist

## Fit-For-Phase Rule

Before choosing a pattern, ask:
- Is this a discovery-stage package, a clinical-stage package, a commercial-stage package, or a post-approval package?
- Does the user need a working draft, a management summary, or a controlled-document precursor?

Choose the smallest artifact that still helps the team make the next decision.
